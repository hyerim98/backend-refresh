## static 중첩 클래스
- 외부 클래스의 인스턴스와 무관
- 외부 클래스의 static 멤버만 접근 가능
- 독립적인 역할 수행 가능

### 특징
- 메모리 구조가 명확함
- 테스트 및 재사용이 상대적으로 쉬움
- "안에 두었을 뿐, 사실상 별도 클래스"

## 내부 클래스 (non-static)
- 외부 클래스의 인스턴스에 소속
- 외부 클래스의 모든 멤버 접근 가능
- 외부 객체 없이는 생성 불가

### 특징
- 외부 객체 상태에 강하게 의존
- 결합도가 높음
- 외부 객체의 구현 세부사항에 접근 가능

## 사용 기준 정리

### static 중첩 클래스를 쓰는 경우
- 외부 클래스와 논리적으로 묶여 있지만
- 외부 인스턴스 상태에 의존하지 않을 때

### 내부 클래스를 쓰는 경우
- 외부 객체의 상태를 자연스럽게 사용해야 할 때
- 외부에서 사용할 필요가 없을 때

## 나의 이해 정리
- 중첩 클래스는 "캡슐화 도구"다
- static 여부는 '외부 인스턴스 의존성' 기준으로 판단한다
- 내부 클래스는 강력하지만 결합도가 높아 남용하면 위험하다
