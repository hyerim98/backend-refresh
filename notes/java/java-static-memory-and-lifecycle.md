# Java static과 메모리 구조, 생명주기 정리

## 정리 목적
- static이 왜 존재하는지 이해한다
- static이 메모리 어디에 저장되는지 설명할 수 있다
- 인스턴스와 static의 생명주기 차이를 명확히 구분한다
- 스프링에서 static 사용을 지양하는 이유의 기초를 만든다

---

## Java 메모리 구조

자바 프로그램 실행 시 메모리는 크게 세 영역으로 나뉜다.

### 1. Method Area (메서드 영역)
- 클래스 정보 저장
- static 변수, static 메서드 저장
- 프로그램 시작 시 클래스 로딩과 함께 생성
- JVM 종료 시까지 유지됨
- **가장 생명주기가 길다**

👉 static 멤버는 모두 Method Area에 존재한다

---

### 2. Stack Area (스택 영역)
- 메서드 호출 시 생성되는 실행 공간
- 지역 변수, 매개변수 저장
- 메서드 호출마다 스택 프레임 생성
- 메서드 종료 시 자동 제거

👉 짧고 빠른 실행용 메모리

---

### 3. Heap Area (힙 영역)
- 객체(인스턴스) 저장
- `new` 키워드로 생성
- 참조가 끊기면 GC 대상
- 생명주기가 유동적

👉 인스턴스는 Heap에 저장된다

---

## static 이란?

> static은 **객체가 아닌 클래스에 소속된 멤버**이다

- 객체 생성 없이 접근 가능
- 클래스당 단 하나만 존재(인스턴스가 몇개 생겨도 메모리에 단 하나만 존재)
- 모든 인스턴스가 같은 static 멤버를 공유한다
- Method Area에 저장된다
