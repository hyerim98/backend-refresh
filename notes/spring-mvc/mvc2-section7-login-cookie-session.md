# Section 07 - 로그인 처리1 (쿠키, 세션)

> 로그인은 "인증 상태를 유지"하는 기술이다.

---

# 1️⃣ 로그인 처리의 문제점

HTTP는 Stateless 프로토콜이다.

- 요청마다 새로운 연결
- 서버는 이전 요청을 기억하지 못함

따라서 로그인 상태를 유지하려면
별도의 저장 수단이 필요하다.

---

# 2️⃣ 쿠키 (Cookie)

## 개념

- 클라이언트(브라우저)에 저장되는 데이터
- 요청 시 자동으로 서버에 전송됨

## 동작 흐름

로그인 성공  
↓  
서버가 쿠키 생성  
↓  
브라우저에 저장  
↓  
이후 요청마다 쿠키 자동 전송

---

## 쿠키 기반 로그인 방식

사용자 로그인  
↓  
회원 ID를 쿠키에 저장  
↓  
브라우저가 요청 시 쿠키 전달  
↓  
서버가 쿠키로 사용자 식별

---

## 쿠키 방식의 문제점

- 쿠키 값은 클라이언트가 수정 가능
- 보안에 취약
- 민감 정보 저장 위험

---

# 3️⃣ 세션 (Session)

## 개념

- 서버에 로그인 정보를 저장
- 클라이언트에는 세션 ID만 저장

---

## 세션 동작 흐름

로그인 성공  
↓  
서버가 세션 생성  
↓  
세션 저장소에 사용자 정보 저장  
↓  
세션 ID를 쿠키로 전달  
↓  
이후 요청 시 세션 ID 전달  
↓  
서버가 세션 저장소에서 사용자 조회

---

# 4️⃣ 세션의 장점

✔ 사용자 정보 서버에 저장  
✔ 보안성 향상  
✔ 쿠키 변조 위험 감소

---

# 5️⃣ HttpSession 사용

- request.getSession()
- 세션 생성 및 조회
- setAttribute()
- getAttribute()
- invalidate() → 로그아웃

---

# 6️⃣ 세션과 쿠키의 관계

쿠키는 세션 ID를 전달하는 수단일 뿐이다.

실제 로그인 정보는 서버 세션 저장소에 존재한다.

---

# 🔥 핵심 정리

로그인 상태 유지는

쿠키에 직접 저장하는 것이 아니라  
세션 ID를 쿠키로 전달하고  
서버 세션에 사용자 정보를 저장하는 방식으로 구현한다.
