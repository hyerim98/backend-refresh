# Section 03 - 타임리프 스프링 통합과 폼

> 타임리프는 스프링과 통합되어  
> HTML Form과 자바 객체를 자동으로 바인딩한다.

---

# 1️⃣ 스프링 폼 바인딩 구조

사용자 입력  
↓  
HTML name 속성  
↓  
HTTP 요청 파라미터  
↓  
@ModelAttribute  
↓  
객체 필드에 자동 바인딩

📌 name 속성과 객체 필드명이 일치해야 한다.

---

# 2️⃣ th:object

- 폼에서 사용할 객체 지정
- 선택 변수 표현식(*{})의 기준이 됨

역할:
- 바인딩 기준 객체 설정
- 폼 전체와 객체 연결

---

# 3️⃣ th:field

- name 자동 생성
- id 자동 생성
- value 자동 바인딩
- 체크 상태 자동 처리

📌 폼 바인딩의 핵심 기능

---

# 4️⃣ 입력 타입별 바인딩

## 4.1 단일 체크박스

- 체크하지 않으면 값이 전송되지 않음(false가 아닌 null이라서 전송 자체가 안됨)
- 이를 해결하기 위해 hidden 필드가 함께 사용됨

📌 체크 해제 상태도 서버에 전달하기 위함

---

## 4.2 다중 체크박스

- 여러 값 선택 가능
- List 또는 배열로 바인딩

---

## 4.3 라디오 버튼

- 하나만 선택 가능
- 단일 값 바인딩

---

## 4.4 셀렉트 박스

- 단일 선택
- 다중 선택 가능 (multiple 옵션 사용)

---

# 5️⃣ th:value vs th:field 차이

th:value
- 단순 값 설정

th:field
- 스프링 바인딩 기능 포함
- 검증 오류와도 자동 연동

---

# 6️⃣ 스프링 통합의 의미

타임리프는 단순 템플릿 엔진이 아니라

✔ 폼 데이터 바인딩 자동화  
✔ 입력값 유지  
✔ 검증과 자연스러운 연결

을 지원하는 View 기술이다.

---

# 🔥 섹션3 핵심 정리

타임리프 스프링 통합은

HTML Form과 자바 객체를  
자동으로 연결하는 기능이다.
