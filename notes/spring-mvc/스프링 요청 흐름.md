# 스프링 요청 흐름

## 정상 요청 흐름
```
Client
→ Servlet Container (Tomcat)
→ Filter
→ FilterChain
→ DispatcherServlet
→ HandlerMapping
→ HandlerExecutionChain 생성
→ Interceptor preHandle
→ HandlerAdapter
→ Controller
→ Service
→ Repository
→ Controller return
→ HandlerAdapter
→ Interceptor postHandle
→ ViewResolver / HttpMessageConverter
→ Response 생성
→ Interceptor afterCompletion
→ Filter
→ Servlet Container
→ Client
```

## API 예외 발생 시 흐름(Spring이 처리하는 경우)
```
Client
 → Servlet Container
 → Filter
 → DispatcherServlet
 → HandlerMapping
 → Interceptor preHandle
 → HandlerAdapter
 → Controller
 → Service
 → ❌ Exception 발생
 → DispatcherServlet catch
 → HandlerExceptionResolver
 → @ControllerAdvice / @ExceptionHandler
 → Response 생성
 → Interceptor afterCompletion
 → Filter
 → Client
```

## 오류 페이지 반환 예외 처리 흐름(Spring이 예외 처리 못한 경우 - 에러 페이지)
```
Client
 → Servlet Container
 → Filter
 → DispatcherServlet
 → HandlerMapping
 → Interceptor preHandle
 → HandlerAdapter
 → Controller
 → Service
 → ❌ Exception 발생
 → DispatcherServlet catch
 → HandlerExceptionResolver
      → 성공 → Response 반환
      → 실패
 → Interceptor afterCompletion
 → Servlet Container
 → ERROR dispatch
 → /error 요청
 → Filter
 → DispatcherServlet
 → HandlerMapping
 → Interceptor preHandle
 → BasicErrorController
 → ErrorViewResolver
 → error.html / JSON
 → Interceptor postHandle
 → Interceptor afterCompletion
 → Client
```
