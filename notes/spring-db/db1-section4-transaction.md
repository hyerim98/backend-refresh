# Section4 - 트랜잭션 이해

## 트랜잭션(Transaction)

트랜잭션은 데이터베이스 작업을 하나의 논리적인 단위로 묶는 것이다.

```
모두 성공 → commit
하나라도 실패 → rollback
```

---

## 트랜잭션이 필요한 이유

예: 계좌이체

```
A 계좌 -10000
B 계좌 +10000
```

문제 상황

```
A 계좌 출금 성공
B 계좌 입금 실패
```

→ 데이터 불일치 발생

---

## 트랜잭션 적용

```
전체 성공 → commit
중간 실패 → rollback
```

→ 데이터 일관성 유지

---

## commit / rollback

### commit

- 트랜잭션 내 작업을 DB에 반영

---

### rollback

- 트랜잭션 내 작업을 모두 취소

---

## 트랜잭션 ACID 특징

### Atomicity (원자성)

- 작업은 모두 성공하거나 모두 실패해야 한다.

---

### Consistency (일관성)

- 트랜잭션 전후 데이터는 일관성을 유지해야 한다.

---

### Isolation (격리성)

- 동시에 실행되는 트랜잭션은 서로 영향을 주지 않는다.

---

### Durability (지속성)

- commit된 데이터는 영구적으로 저장된다.

---

## JDBC 트랜잭션 처리

기본적으로 JDBC는 자동 커밋 모드이다.

```
autoCommit = true
```

→ SQL 실행 시 자동으로 commit

---

## 자동 커밋 OFF

트랜잭션을 직접 제어하려면

```
autoCommit = false
```

설정이 필요하다.

---

## JDBC 트랜잭션 흐름

```
Connection 획득
 ↓
autoCommit(false)
 ↓
비즈니스 로직 실행
 ↓
성공 → commit
실패 → rollback
 ↓
Connection 종료
```

---

## 트랜잭션 경계

트랜잭션은 보통 비즈니스 로직 단위로 설정한다.

```
Controller
 ↓
Service (트랜잭션 시작)
 ↓
Repository
 ↓
DB
```

---

## 핵심 개념 ⭐⭐⭐⭐⭐

```
트랜잭션은 하나의 작업 단위이다
```

---

## 주의사항

- 트랜잭션 범위를 너무 크게 잡으면 성능 저하
- 너무 작으면 데이터 일관성 깨질 수 있음

---

## 한 줄 정리

```
트랜잭션은 데이터의 일관성을 보장하기 위한 작업 단위이다
```

---

## 로우 락(Row Lock)

데이터베이스는 트랜잭션 간 데이터 충돌을 방지하기 위해  
특정 데이터(행, Row)에 락을 건다.

```
트랜잭션 사용 시 자동으로 로우에 락 걸리는 SQL문

UPDATE, DELETE, 
SELECT FOR UPDATE(좌석, 재고, 잔액 확인 후 차감 로직은 로우 조회 시 락 필요)
```

---

## 락이 필요한 이유

동시에 여러 트랜잭션이 같은 데이터를 수정할 경우

예

```
트랜잭션1 → 잔액 -10000
트랜잭션2 → 잔액 -5000
```

→ 동시에 실행되면 데이터가 꼬일 수 있다.

---

## 로우 락 동작

트랜잭션이 데이터를 수정하면  
해당 행(Row)에 락이 걸린다.

```
트랜잭션1
 ↓
특정 Row 수정
 ↓
Row Lock 획득
```

---

## 동시 요청 상황

```
트랜잭션1 → Row A 수정 (락 획득)
트랜잭션2 → Row A 수정 시도
```

결과

```
트랜잭션2 → 대기 (Blocking)
```

---

## 락 해제 시점

락은 트랜잭션이 끝날 때 해제된다.

```
commit → 락 해제
rollback → 락 해제
```

---

## 핵심 특징

- 락은 트랜잭션 단위로 동작
- 같은 데이터에 동시에 접근 불가
- 데이터 일관성 보장

---

## 주의사항

트랜잭션이 길어지면

```
락 유지 시간 증가
 → 다른 트랜잭션 대기
 → 성능 저하
```

---

## 한 줄 정리

```
트랜잭션은 데이터를 수정할 때 Row에 락을 걸어 동시성 문제를 방지한다

트랜잭션 = Connection 단위 / 작업 단위 관리
락 = 데이터(Row) 단위 / 데이터 충돌 방지
```
