## 실습 목적
- static 중첩 클래스와 내부 클래스 생성 방식 비교
- 접근 가능한 멤버 차이 확인
- 컴파일 에러가 발생하는 경우 직접 확인

## static 중첩 클래스

### 확인한 점
- 외부 클래스 인스턴스 없이 생성 가능
- 외부 클래스의 static 멤버만 접근 가능
- 내부적으로 독립된 클래스처럼 동작
- 바깥 클래스의 인스턴스에 소속되지 않는다

## 내부 클래스 (non-static)

### 확인한 점
- 외부 클래스 인스턴스 없이는 생성 불가
- 외부 클래스의 모든 멤버 접근 가능
- 외부 객체와 강하게 결합됨

## 실습을 통해 느낀 점
- 내부 클래스는 편하지만 구조를 복잡하게 만든다
- static 중첩 클래스가 더 안전한 기본 선택지로 느껴진다
